# Agile Product Development & Scrum

## Overview

This project demonstrates the application of Agile software development principles and the Scrum framework throughout the software development lifecycle. The focus was on understanding Agile team roles, managing changing requirements, creating user stories, refining product backlogs, and supporting iterative product delivery through Scrum ceremonies and continuous collaboration.

## Business Problem

Software projects often face changing requirements, evolving customer expectations, and the need for frequent delivery of working solutions. This project explored how Scrum and Agile practices enable development teams to respond to change, improve collaboration, and deliver value through iterative development.

## My Role

- Applied Scrum and Agile principles throughout the software development lifecycle
- Analyzed Product Owner, Scrum Master, and Developer responsibilities
- Developed and evaluated user stories and product backlog refinement
- Examined sprint planning, daily Scrum meetings, and retrospectives
- Evaluated Agile practices for managing changing requirements and continuous improvement
- Documented project outcomes and Agile development processes

This project strengthened my understanding of how Agile teams plan, develop, test, and continuously improve software through iterative delivery. It reinforced the importance of stakeholder collaboration, backlog prioritization, adaptability to changing requirements, and effective communication throughout the software development lifecycle.
